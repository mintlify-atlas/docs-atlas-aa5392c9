---
title: 'Get Activity Feed'
api: 'GET /api/tx/activity'
description: 'Retrieve filtered transaction activity for the authenticated user addresses'
---

## Authentication

This endpoint requires authentication. Include your session token in the request headers.

## Query Parameters

<ParamField query="type" type="string" default="all">
  Filter transactions by type:
  - `"all"` - All transactions where user is the sender
  - `"payments"` - Confirmed transactions from user addresses
  - `"sends"` - All transactions from user addresses (any status)
</ParamField>

<ParamField query="limit" type="number" default={50}>
  Number of items to return (max 100)
</ParamField>

<ParamField query="offset" type="number" default={0}>
  Number of items to skip for pagination
</ParamField>

## Response

<ResponseField name="items" type="array">
  Array of transaction activity items
  
  <ResponseField name="id" type="number">
    Transaction ID
  </ResponseField>
  
  <ResponseField name="type" type="string">
    Activity type: `"payment"` (confirmed outgoing) or `"send"` (any outgoing)
  </ResponseField>
  
  <ResponseField name="hash" type="string">
    Transaction hash
  </ResponseField>
  
  <ResponseField name="chainId" type="number">
    Chain ID where the transaction occurred
  </ResponseField>
  
  <ResponseField name="fromAddress" type="string">
    Sender's wallet address
  </ResponseField>
  
  <ResponseField name="toAddress" type="string">
    Recipient's wallet address
  </ResponseField>
  
  <ResponseField name="value" type="string">
    Transaction amount
  </ResponseField>
  
  <ResponseField name="tokenSymbol" type="string">
    Token symbol (e.g., "USDC", "ETH")
  </ResponseField>
  
  <ResponseField name="status" type="string">
    Transaction status: `"pending"`, `"confirmed"`, or `"failed"`
  </ResponseField>
  
  <ResponseField name="createdAt" type="string">
    ISO 8601 timestamp
  </ResponseField>
</ResponseField>

<ResponseField name="total" type="number">
  Total number of matching transactions (before pagination)
</ResponseField>

## Example Request

<CodeGroup>
```bash cURL
curl -X GET "https://api.walty.com/api/tx/activity?type=payments&limit=20&offset=0" \
  -H "Cookie: session=your_session_token"
```

```javascript JavaScript
const response = await fetch(
  'https://api.walty.com/api/tx/activity?type=payments&limit=20&offset=0',
  {
    headers: {
      'Cookie': 'session=your_session_token'
    }
  }
);
const data = await response.json();
console.log(`Found ${data.total} transactions`);
```

```python Python
import requests

response = requests.get(
    'https://api.walty.com/api/tx/activity',
    params={'type': 'payments', 'limit': 20, 'offset': 0},
    cookies={'session': 'your_session_token'}
)
data = response.json()
print(f"Found {data['total']} transactions")
```
</CodeGroup>

## Filter Behavior

<Note>
  This endpoint automatically fetches all addresses associated with the authenticated user and filters transactions accordingly. If the user has no addresses, it returns an empty result.
</Note>

### Type Filters

- **`all`**: Returns all transactions where any of the user's addresses is the sender
- **`payments`**: Returns only confirmed transactions from user addresses
- **`sends`**: Returns all outgoing transactions from user addresses (pending, confirmed, or failed)

<Warning>
  The maximum limit per request is 100. Requests exceeding this will be capped at 100 items.
</Warning>
