# Payment Requests

Walty's payment request system enables businesses to accept cryptocurrency payments through QR codes and shareable links. Built for point-of-sale scenarios, it includes automatic on-chain reconciliation, expiration handling, and split payment support.

## Overview

Payment requests are designed for merchant use cases:

- **QR Code Generation** - Instant payment links customers can scan
- **Automatic Expiration** - Requests expire after 15 minutes by default
- **On-Chain Monitoring** - Automatically detects incoming payments
- **Confirmation Tracking** - Waits for required confirmations (2 by default)
- **Split Payments** - Multiple customers can contribute to a single request

<Info>
  Payment requests currently support USDC and USDT on Polygon (Chain ID: 137) for low fees and fast confirmations.
</Info>

## Creating Payment Requests

Business accounts can create payment requests through the dashboard:

  </Step>

  <Step title="Generate Request">
    The server creates a payment request with:
    - Unique request ID
    - Expiration timestamp (15 minutes)
    - Merchant wallet address
    - Required confirmations (2)

    ```typescript

    {
      "id": "pay_abc123def456",
      "status": "pending",
      "chainId": 137,
      "amountUsd": "25.00",
      "tokenSymbol": "USDC",
      "merchantWalletAddress": "0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb",
      "expiresAt": "2026-03-14T12:30:00.000Z",
      "requiredConfirmations": 2
    }
    ```
  </Step>

  <Step title="Display QR Code">
    Generate a QR code encoding the payment URL:

    ```typescript
    const paymentUrl = `https://app.walty.io/pay/${requestId}`
    
    <QRCodeSVG 
      value={paymentUrl} 
      size={200} 
      level="M" 
    />
    ```

  </Step>

  <Step title="Monitor Status">

    ```typescript
    GET /api/payment-requests/{requestId}
    
    // Status transitions:
    // pending → confirming → paid
    // pending → expired
    ```
  </Step>
</Steps>

## Countdown Timer

Payment requests show real-time countdown:

```typescript
export function getPaymentRequestCountdown(
  expiresAt: string,
  now: number = Date.now()
): { expired: boolean; label: string; seconds: number } {
  const seconds = Math.max(
    0, 
    Math.floor((new Date(expiresAt).getTime() - now) / 1000)
  )
  
  const minutesLabel = Math.floor(seconds / 60)
    .toString()
    .padStart(2, "0")
  const secondsLabel = (seconds % 60)
    .toString()
    .padStart(2, "0")

  return {
    expired: seconds === 0,
    label: `${minutesLabel}:${secondsLabel}`,
    seconds,
  }
}
```

**Usage:**

```tsx
const [now, setNow] = useState(Date.now())

useEffect(() => {
  const clockId = setInterval(() => setNow(Date.now()), 1000)
  return () => clearInterval(clockId)
}, [])

const countdown = getPaymentRequestCountdown(request.expiresAt, now)

<span className="font-mono text-xs">
  {countdown.expired 
    ? "Expired" 
    : `Expires in ${countdown.label}`
  }
</span>
```

## Automatic Reconciliation

Walty's backend automatically monitors the blockchain for payments:

```typescript
// Runs every 30 seconds via cron job
export const PAYMENT_RECONCILE_INTERVAL_SECONDS = 30

// Reconciliation process:
// 1. Fetch all pending/confirming payment requests
// 2. Query blockchain for transactions to merchant addresses
// 3. Match transactions to requests by:
//    - Recipient address
//    - Token (USDC/USDT)
//    - Amount (within tolerance)
//    - Timestamp (after request creation)
// 4. Update request status and confirmations
// 5. Mark as paid when confirmations >= requiredConfirmations
```

**Configuration:**

```typescript
export const PAYMENT_CHAIN_ID = 137  // Polygon
export const PAYMENT_EXPIRY_MINUTES = 15
export const PAYMENT_REQUIRED_CONFIRMATIONS = 2
export const PAYMENT_ALLOWED_TOKENS = ["USDC", "USDT"] as const
```

## Payment Status Helpers

```typescript
export function getPaymentRequestStatus(
  request: Pick<PaymentRequestView, "status" | "expiresAt">,
  now: number = Date.now()
): PaymentRequestStatus {
  if (request.status === "pending" && 
      new Date(request.expiresAt).getTime() <= now) {
    return "expired"
  }
  return request.status
}

export function isPaymentRequestActive(
  request: Pick<PaymentRequestView, "status" | "expiresAt">,
  now: number = Date.now()
): boolean {
  const status = getPaymentRequestStatus(request, now)
  return status === "pending" || status === "confirming"
}
```

## Confirmation Tracking

Payments require blockchain confirmations before being marked as paid:

```tsx
<div className="flex items-center justify-between">
  <div>
    <p className="text-xs">Status</p>
    <p className="text-sm font-medium">
      {status === "confirming" 
        ? "Payment Detected"
        : getPaymentRequestStatusLabel(status)
      }
    </p>
  </div>
  {status === "confirming" && (
    <span className="font-mono text-xs text-amber-600">
      {request.confirmations}/{request.requiredConfirmations} confirmations
    </span>
  )}
</div>
```

<Note>
  Polygon confirmations are typically very fast (2-5 seconds each). Most payments complete within 30 seconds.
</Note>

## Polling Intervals

Different polling rates for different contexts:

```typescript
// Active payment page (customer/merchant viewing)
export const PAYMENT_MODAL_POLL_INTERVAL_MS = 3_000  // 3 seconds

// Dashboard list view
export const PAYMENT_HOME_POLL_INTERVAL_MS = 5_000   // 5 seconds

// Backend reconciliation
export const PAYMENT_RECONCILE_INTERVAL_SECONDS = 30  // 30 seconds
```

**Implementation:**

```typescript
useEffect(() => {
  if (!request) return
  if (request.status !== "pending" && request.status !== "confirming") return

  const poll = async () => {
    const res = await fetch(`/api/payment-requests/${requestId}`)
    if (res.ok) setRequest(await res.json())
  }

  const pollId = setInterval(poll, PAYMENT_MODAL_POLL_INTERVAL_MS)
  return () => clearInterval(pollId)
}, [request, requestId])
```

## Error States

Handle various edge cases:

```tsx
// Request not found
if (!request) {
  return (
    <div className="text-center">
      <p className="text-lg font-semibold">
        This payment request is not available
      </p>
      <p className="text-sm text-muted-foreground">
        Request a new QR code from the merchant.
      </p>
    </div>
  )
}

// Already paid
if (status === "paid") {
  return (
    <div className="text-center">
      <p className="text-lg font-semibold">
        This payment has already been completed
      </p>
    </div>
  )
}

// Expired
if (status === "expired") {
  return (
    <div className="text-center">
      <p className="text-lg font-semibold">
        This payment request has expired
      </p>
      <p className="text-sm text-muted-foreground">
        Request a new QR code from the merchant.
      </p>
    </div>
  )
}
```
