---
title: 'Record Transaction'
api: 'POST /api/tx'
description: 'Record a new transaction in the system (always starts as pending)'
---

## Authentication

This endpoint requires authentication. Include your session token in the request headers.

## Request Body

<ParamField body="hash" type="string" required>
  Transaction hash (unique identifier)
</ParamField>

<ParamField body="chainId" type="number" default={1}>
  Chain ID where the transaction occurred (defaults to 1 for Ethereum mainnet)
</ParamField>

<ParamField body="chainType" type="string" default="EVM">
  Type of blockchain (defaults to "EVM")
</ParamField>

<ParamField body="fromAddress" type="string" required>
  Sender's wallet address
</ParamField>

<ParamField body="toAddress" type="string" required>
  Recipient's wallet address
</ParamField>

<ParamField body="tokenAddress" type="string">
  Token contract address (null for native tokens like ETH)
</ParamField>

<ParamField body="tokenSymbol" type="string" required>
  Token symbol (e.g., "ETH", "USDC")
</ParamField>

<ParamField body="value" type="string" required>
  Transaction amount (formatted with decimals)
</ParamField>

## Response

<ResponseField name="ok" type="boolean">
  Indicates successful transaction recording
</ResponseField>

## Example Request

<CodeGroup>
```bash cURL
curl -X POST "https://api.walty.com/api/tx" \
  -H "Content-Type: application/json" \
  -H "Cookie: session=your_session_token" \
  -d '{
    "hash": "0x1234567890abcdef",
    "chainId": 1,
    "chainType": "EVM",
    "fromAddress": "0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb",
    "toAddress": "0x8ba1f109551bD432803012645Ac136ddd64DBA72",
    "tokenAddress": "0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48",
    "tokenSymbol": "USDC",
    "value": "100.50"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.walty.com/api/tx', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',

  },
  body: JSON.stringify({
    hash: '0x1234567890abcdef',
    chainId: 1,

    fromAddress: '0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb',
    toAddress: '0x8ba1f109551bD432803012645Ac136ddd64DBA72',
    tokenAddress: '0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48',
    tokenSymbol: 'USDC',
    value: '100.50'
  })
});
const data = await response.json();
```
</CodeGroup>

## Example Response

```json
{
  "ok": true
}
```

## Error Responses

<ResponseField name="error" type="string">
  Error message
</ResponseField>

<CodeGroup>
```json 400 Bad Request
{
  "error": "Missing fields"
}
```

```json 401 Unauthorized
{
  "error": "Unauthorized"
}
```
</CodeGroup>

<Note>
  All transactions are recorded with an initial status of `"pending"`. Use the PATCH endpoint to update the status to `"confirmed"` or `"failed"` once the transaction is finalized on-chain.
</Note>

<Warning>
  The `hash`, `fromAddress`, `toAddress`, `tokenSymbol`, and `value` fields are required. Missing any of these will result in a 400 error.
</Warning>
